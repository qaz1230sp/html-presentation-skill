    Agent Skill 入门指南

    让 AI 助手学会新技能

    Speaker · 2025

    核心价值

    可复用

    把成熟做法沉淀成固定能力，后续场景直接复用。

    自动触发

    写清描述与触发词后，模型会在合适时机主动调用。

    组件化

    规则、模板、脚本与示例可以拆分维护、按需组合。

    社区共享

    团队经验不再只留在聊天里，而能被持续传播与协作。

    什么是 Agent Skill？

    一句话定义

    Skill 是给 AI 助手写的操作手册

    • 告诉助手什么时候该出手
    • 规定步骤、边界与输出格式
    • 把经验从“会做”变成“稳定复现”
    
---
name: demo-skill
description: 当用户要生成日报时触发
---

## Steps
1. 收集输入
2. 套用模板
3. 输出结果

    Skill vs 传统方式

    传统方式

    • 每次重写 Prompt
    • 需要编码 Plugin
    • 静态配置文件
    • 无法跨项目复用

    Agent Skill

    • 纯 Markdown 编写
    • 自动场景匹配
    • 上下文感知
    • Git 版本管理

    Skill 文件结构

    • SKILL.md 是唯一必需文件
    • Frontmatter 定义触发条件
    • 正文描述操作流程
    • 可附带脚本和模板
    
# my-skill
- SKILL.md
- templates/
  - report-template.md
- scripts/
  - validate.py
- examples/
  - demo-input.json

    创建流程

    1

    确定场景

    明确 Skill 要解决的具体问题和使用场景

    2

    编写 SKILL.md

    定义触发词、工作流工具调用与输出格式

    3

    测试验证

    多场景触发测试优化描述与输出质量

    4

    发布共享

    推送到 GitHub 仓库供团队或社区安装使用

    先让触发清晰，再让步骤稳定，最后再推广复用。

    Skill 生态增长

    示例数据 · 社区目录观察；工具类与报告类增长最快。

    Skill 使用场景分布

    示例数据 · 使用场景抽样；代码辅助仍是主力，文档与分析紧随其后。

    最佳实践速查

    写 Skill 前，先用这张表做一次快速自检。

    维度 ✅ 推荐做法 ❌ 避免做法
    命名 动词短语，如“生成周报” 只写名词，如“周报”
    Description 写清触发条件与上下文 只写笼统功能名
    篇幅 聚焦核心步骤，控制在 <500 词 把所有内容都塞进一个文件
    测试 先看无 Skill 表现，再比较增益 直接部署，不做基线验证

    把命名、触发、篇幅、测试四件事先做对，Skill 成功率会明显提高。
