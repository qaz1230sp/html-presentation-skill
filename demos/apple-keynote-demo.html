    核心价值

    可复用

    把成熟做法沉淀成固定能力，后续场景直接复用。

    自动触发

    写清描述与触发词后，模型会在合适时机主动调用。

    组件化

    规则、模板、脚本与示例可以拆分维护、按需组合。

    社区共享

    团队经验不再只留在聊天里，而能被持续传播与协作。

    什么是 Agent Skill？

    定义

    Agent Skill 是给 AI 助手准备的可复用能力包。

    • 用 Markdown 描述场景、规则与执行步骤
    • 通过描述自动匹配，而不是每次重新写长 Prompt
    • 可以封装模板、脚本、图表、文档读取等工作流
    把经验写成技能，而不是留在一次次对话里。

    Agent Skill

    • 上下文感知

    Skill 文件结构

    • 前置 metadata 决定名称、触发范围与能力定位
    • Overview 说明用途，帮助助手快速选对技能
    • Workflow 约束步骤，让输出更稳定、更可复现
    • Reference 与 Checklist 保证组件复用与质量校验

## Quality Checklist

    创建流程

    1

    确定场景

    明确 Skill 要解决的具体问题和使用场景

    2

    编写 SKILL.md

    定义触发词、工作流工具调用与输出格式

    3

    测试验证

    多场景触发测试优化描述与输出质量

    4

    发布共享

    推送到 GitHub 仓库供团队或社区安装使用

    从真实场景出发，先做可用版本，再通过验证不断打磨触发与输出质量。

    Skill 生态增长

    示例数据 · 工具类领先，报告类与创作类持续增长，生态正向多场景扩展。

    Skill 使用分布

    示例数据 · 代码辅助仍是主力入口，但文档、分析与自动化需求正在形成稳定第二曲线。

    最佳实践

    让 Skill 更易触发、更好维护、更容易复用

    维度 ✅ 推荐 ❌ 避免
    命名 短、准、可搜索，如 weekly-report 模糊抽象，如 helper123
    描述 明确场景、输入与触发词 只写“处理任务”之类空泛描述
    长度 结构清晰，分节表达规则与步骤 所有要求堆成一大段长文
    测试 用真实样例反复验证触发与输出 写完即发布，从不回看效果

    经验越可验证，Skill 越容易长期沉淀成团队资产。
